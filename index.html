    מושגי יסוד ומלכודות נפוצות

    בקרת גישה (Unix & Windows)

    הספרות הן 4 (R), 2 (W), 1 (X).

    "רוצה ורץ" (R-W-X) - קריאה, כתיבה, ריצה.
    • SUID (4xxx): מריץ בכוח של הבעלים.
    • SGID (2xxx): מריץ עם הרשאת הקבוצה.
    • Sticky Bit (1xxx): רק הבעלים מוחק.
    מלכודת ה-ACL: ביוניקס בסיסי אין דרך לתת למשתמשים שונים הרשאות שונות על אותו קובץ בלי קבוצות מורכבות.

    מודלים (BLP vs Biba)

    מודל מטרה כלל
    BLP סודיות No Read Up, No Write Down
    Biba שלמות No Read Down, No Write Up
    BLP: "לא קורא למעלה, לא כותב למטה" - מונע דליפת מידע.

    אנטרופיה וזרימת מידע

    נוסחה: log₂(אפשרויות לפני / אפשרויות אחרי)

    דוגמה: 8 אפשרויות ל-2 אפשרויות = log₂(4) = 2 ביטים דלפו.

    יותר אפשרויות לפני = יותר אנטרופיה אבדה.
    מלכודת הלוגריתם: אנטרופיה נמדדת בלוגריתם, לא בהפרש רגיל!

    פגיעויות תוכנה ומנגנוני הגנה

    הגנה מה זה עושה?
    DEPחוסם הרצה מהמחסנית.
    ASLRמגריל כתובות בזיכרון.
    Canaryמזהה גלישה לפני פריצה.
    NX Bitמסמן זיכרון כלא-ניתן-להרצה.

    קריפטוגרפיה - הצפנה סימטרית

    • AES: Advanced Encryption Standard - 128/192/256 ביט.
    • DES: הוחלף ע"י AES (56 ביט מפתח בלבד).
    • ECB Mode: הצפנה בלוק בלוק - חשוף דפוסים!
    • CBC Mode: כל בלוק מקושר לקודם (IV נדרש).
    מלכודת ה-ECB: ECB תמיד נותן פלט זהה לאותו קלט - חושף דפוסים!

    קריפטוגרפיה - הצפנה אסימטרית

    • RSA: מבוסס על פירוק מספרים לגורמים ראשוניים.
    • Diffie-Hellman: הסכמת מפתחות - יוצר מפתח משותף.
    • Digital Signature: חתימה עם המפתח הפרטי, אימות עם הציבורי.
    RSA = סימטרי לאסימטרי, אסימטרי להחלפת מפתחות.

    💡 טיפים והוקוסים - שאלות חוזרות במבחנים

    שאלות שמופיעות הרבה ושוב במבחנים - כדאי להכיר היטב!

    🔴 מודלי BLP ו-Biba - שאלות חוזרות

    שאלה נפוצה (מופיעה ב-91, 92, מועד א1, מועד א2): נתונה מערכת עם משתמשים וקבצים ברמות סיווג שונות.
    השאלה: האם משתמש X יכול לקרוא/לכתוב לקובץ Y?
    הטיפ: זכור: BLP = "לא קורא למעלה, לא כותב למטה" | Biba = "לא קורא למטה, לא כותב למעלה"
    • BLP: Simple Security = No Read Up | *-Property = No Write Down
    • Biba: Simple Integrity = No Read Down | *-Integrity = No Write Up
    • מבחן: רמות נמוכות במספר = רמה גבוהה יותר (TS=0, Secret=1, C=2, U=3)

    📋 ייצוג מטריצת גישה (מופיע ב-91, 92)

    שאלה נפוצה: האם ניתן לייצג מטריצת גישה ב-UNIX (9 ביטים)? הראו כיצד.
    התשובה: לא ב-UNIX רגיל (יש רק rwx לבעלים/קבוצה/אחרים). אפשר ב-Windows עם ACL.
    • ACL: רשימה לכל אובייקט - מי רשאי לגשת
    • C-List: רשימת יכולות לכל סובייקט - לאילו אובייקטים יש גישה
    • Access Matrix: טבלה של כל הסובייקטים × כל האובייקטים

    🔐 RSA חתימה דיגיטלית (מופיע ב-91, 92, מועד א1)

    שאלת חתימה (נפוצה מאוד): אליס רוצה לשלוח הודעה M לבוב ולחתום עליה.
    השאלה: כיצד בוב מאמת שההודעה מאליס?
    התשובה: אליס חותמת עם המפתח הפרטי שלה: S = M^d mod n. בוב בודק עם המפתח הציבורי של אליס: M = S^e mod n.
    • חתימה: הודעה + מפתח פרטי → חתימה
    • אימות: חתימה + מפתח ציבורי → הודעה מקורית
    • דוגמה: אם e=3, d=?. כל M מותאם עם S=M^3, בודקים S^?=M

    🛡️ הגנות מפני Buffer Overflow (מופיע ב-91, 92, מועד א1, מועד א2)

    שאלה נפוצה: תנו דוגמה לפגיעות Buffer Overflow ואיזה מגנוני הגנה ימנעו אותה.
    התשובה (בחר נכון): ASLR, DEP/NX, Canary, Stack Randomization
    הגנה מה היא מונעת
    DEP/NXהרצת קוד מה-Stack
    ASLRניחוש כתובות זיכרון
    Canaryגלישת Stack

    🔐 Hash ו-HMAC (מופיע ב-91, 92, מועד א1)

    שאלה נפוצה: מערכת משתמשת בפונקציית גיבוב t למשלוח הודעות.
    הסבירו כיצד השיטה נפרצת וכיצד HMAC פותרת זאת.
    הבעיה: התוקף יכול לשנות M→M' ולחתום עליה (M' לא משתנה). הפתרון: HMAC = Hash(key + Hash(key + message)) - דורש מפתח סודי!
    • Hash רגיל: צד ג' יכול לזייף (אין מפתח)
    • HMAC: מאמת גם שלמות וגם מקור (דורש מפתח סודי)
    • MD5: לא בטוח - יש התנגשויות (collisions)

    🔥 חומת אש - כתיבת חוקים (מופיע ב-91, 92)

    שאלת כתיבת חוקים (Stateless Firewall): כתוב את טבלת החוקים המתאימה.
    התשובה: ALLOW OUTBOUND TCP port 25 (SMTP), 80 (HTTP), 443 (HTTPS)
    • Stateless: בודק כל חבילה בנפרד, ללא זיכרון
    • Stateful: זוכר מצב חיבורים (מאפשר תעבורה חוזרת)
    • Proxy: בודק גם את תוכן היישום (Application Layer)

    🎫 Kerberos (מופיע ב-91, 92, מועד א1)

    שאלת Kerberos (נפוצה): הציגו את פרוטוקול Kerberos להתחברות לשרת קבצים.
    השלבים: 1. בקשת TGT מה-AS → 2. בקשת Service Ticket מה-TGS → 3. התחברות לשרת
    • AS: Authentication Server - מאמת משתמשים, מעניק TGT
    • TGS: Ticket Granting Server - מעניק Service Tickets
    • TGT: Ticket Granting Ticket - מאפשר קבלת כרטיסי שירות
    • יתרון: סיסמה לא עוברת ברשת (הסיסמה אינה עוברת בטקסט)

    ⚠️ דברים שסטודנטים מתבלבלים בהם - שימו לב!

    • ב-BLP: רמה 0 (TS) גבוהה מרמה 3 (U) - אל תתבלבל!
    • RSA: חתימה = מודולו d (פרטי), אימות = מודולו e (ציבורי)
    • ECB Mode: אותו קלט = אותו פלט → חושף דפוסים! תמיד בחר CBC
    • Hash vs MAC: Hash פשוט לא מאמת מקור, MAC כולל מפתח (מאמת גם מקור)
    • Salt: מונע Rainbow Tables אבל לא Brute Force! (מונע הצפנה מהירה)
    • IDS vs IPS: IDS רק מתריע, IPS חוסם תעבורה זדונית
    • Stateless vs Stateful: Stateless בודק כל חבילה בנפרד, Stateful זוכר חיבורים
    • Zero-day: פגיעות שלא ידועה (אין Patch) - הכי מסוכנת!
