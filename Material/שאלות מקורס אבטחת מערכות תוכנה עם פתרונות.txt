Preimage Resistance:
 
A hash function is preimage resistant if, given a hash value h, it is hard to find any message m
 
such that h = hash(k, m), where k is the hash key.
 
Collision Resistance
 
A hash function is collision resistant if, given two messages m1 and m2, it is hard to find a hash
 
h such that h = hash(k, m1) = hash(k, m2), where k is the hash key.
 
Second Preimage Resistance
 
A hash function is second preimage resistant if given a message m1, it is hard to find a different
 
message m2 such that hash(k, m1) = hash(k, m2), where k is the hash key.

central repository, the certificate validation is less efficient (since it requires network work) and it
 
the central repository is down the entire company network will be down as well.

Answers:
 
 
Sticky bit
 
In Unix systems if a user is given permission to modify a folder he is able to add and delete files
 
from that folder.
 
That creates a challenge if you want to enable users adding files to a folder, but not deleting
 
other users files.
 
For that end, a sticky bit was added allowing users only to delete the files which they own
 
(owner of the folder is not affected)
 
 
Permission elevation in Windows
 
 
In windows a mechanism called UAC was introduced to enable users to run on a low security
 
level even if the user has higher security rights.
 
The user, when performing an administrative action can elevate their permission temporarily
 
using the UAC mechanism.
 
 
Permission elevation in Unix/Linux
 
In Unix/Linux SUID, SGUID are used to allow users to run processes which requires elevated
 
permissions.
 
The process once executed, will run under the owner privileges.
 
 
RM (Reference Moinitor)
 
Reference monitor is the component which bridges between the subjects and objects. It is
 
responsible to check permissions and grant or deny access and also to invoke the auditing
 
system to audit the operation.
 
 
TOCTOU
 
Time of check, Time of use.
 
The concept of when the permission is checked and when the operation is performed.
 
Some attacks utilize the difference between the two to perform an operation they are not
 
supposed to be allowed to do.
 
This is also called a critical section issue, because the mechanism that is responsible for the
 
check is granting the access but then something changes before it can re-evaluate and the
 
attacker performs the action.
 
 
Least Privileged
 
The concept dictates that each participant in a system will be granted the minimal permissions
 
required for that participant to perform his duties but nothing more.
 
In linux systems there are a few limitations to make following this very challenging:
 
1.
The root user is granted access for everything. For example this contradicts this because
 
the root user should administrator the system but not have access to sensitive user data
 
2.
The file permissions in linux are configured as [user] [group] [other] making it very
 
inflexible fine tuning the permissions to reach a state of least privilege.
 
3.
Most linux OS services runs under root privileges granting them full control of the system
 
4.
The linux Kernel is one monolite therefore all the code in the Kernel has full access to
 
everything
 
 
 
Capability
 
A capability is like a permission only represented as a “token to do something”. Unlike in ACL, a
 
capability can be easily delegated to someone else.
 
 
Capability List Adv. over ACL
 
When using Cap list there are a few benefits:
 
1.
Checking the permissions is easy, so the permission can be checked every time it is
 
required
 
2.
Delegating permissions is easy
 
 
3.
Delegating just a subset of the permissions is also easy
 
 
High Water Mark
 
High water mark is a concept best suited for runtime validation. The access level is determined
 
as the maximum level of all the data accessed during the specific session.
 
Since OS needs to read and write to both critical files, and low risk files, if we implement the
 
concept of high-water  mark in compilation time for the OS, all the files will be deemed critical
 
making this unusable.
 
 
TCB
 
TCP is the collection of all the mechanisms in the operating system that enforces the security of
 
the system.

Tor is a decentralized network consisting of a collection of servers called "Nodes". The user
 
data will flow from an entry node, a minimum of one middle node (or more) and finally an exit
 
node. As the route selected randomly on a diverse network, tracking a specific user becomes
 
almost impossible. Tor protects the identity and location of the user, the traffic destination and
 
the traffic itself.
 
Tor highly depends on encryption as the bases for the anonymity it provides. Encryption is
 
utilized at every layer.While the data traverse from each node to the next, a layer of encryption
 
is decrypted and removed. Each node can decrypt only one layer of defense and not the full
 
stack of layers, protecting the core message. It is up to the exit node to peel the last layer of
 
encryption and to hand the core message to the destination. Tor can provide service to clients
 
using the Tor browser, a project based on Mozilla firefox codebase or by other means or
 
software components. Tor also provides services to host web services via hidden addresses
 
called onion addresses, many of them associated with the DarkWeb.

Using IDS as an additional layer of secure on top of using a firewall is recommended.
 
The reason being that a firewall usually only analyzes common information on the lower layers
 
of the network, while an IDS can analyze much more detailed information, signatures of
 
application layer packets, and even employ anomaly detection algorithms.
 
Another reason IDS is useful is that we may allow a higher rate of false positives when using
 
IDS, while in Firewall and IPS systems such false positives may harm the system.
 
Nowadays firewalls may have IDS and IPS functionality already built in.
 
 
Utilizing both signature and anomaly based detections in IDS is beneficial. The first will detect
 
known vulnerabilities and may have lower rate of false positives but higher rate of false
 
negatives (especially when it comes to 0-days).
 
The behavioral IDS will detect 0-days more efficiently but will have more false positives then the
 
signature based and requires more configuration and fine-tuning.
